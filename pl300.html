A practical 4-week plan

  1. Week 1 — Prepare data (hands-on): load CSV/SQL/folder, fix types, unpivot, merge vs append, parameters, query folding. Build the muscle in Power Query, not theory. (See Handbook · Power Query.)
  2. Week 2 — Model + DAX: build a star schema, mark a date table, write measures (SUMX, CALCULATE, time intelligence), set up RLS. Drill the DAX handbook until evaluation context clicks.
  3. Week 3 — Visualize & analyze: rebuild a real report — interactions, slicers, bookmarks, drillthrough, tooltips, conditional formatting, accessibility, key influencers / decomposition tree.
  4. Week 4 — Deploy + mock exams: publish to a workspace, build an app, set scheduled refresh + gateway, apply sensitivity labels. Then take this mock repeatedly until you score 85%+, reviewing every miss.
Pass fasterBuild one real end-to-end report (ingest → model → DAX → visuals → publish → RLS). PL-300 rewards doers. Then practice out loud with the AI interviewer and grind the question bank.
